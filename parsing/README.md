# parsing/

Директория для парсинга интернет-источников.

Здесь размещаются скрипты для сбора данных из открытых интернет-источников (новостные сайты, форумы, блоги и пр.) с помощью веб-скрейпинга.

## Рекомендуемая структура

```
parsing/
├── README.md               ← этот файл
├── scrapers/               ← отдельные скрейперы по источнику
│   ├── base_scraper.py     ← абстрактный базовый класс
│   ├── rbc_scraper.py
│   ├── kommersant_scraper.py
│   └── ...
├── models/                 ← dataclass / Pydantic модели результатов парсинга
│   └── article.py
├── scheduler.py            ← планировщик запуска скрейперов (APScheduler / celery)
└── utils.py                ← вспомогательные функции (очистка HTML, нормализация дат)
```

## Интеграция с backend

Результаты парсинга передаются в `backend` через:
- **Прямую запись в БД** — скрейпер напрямую создаёт записи в таблице `news` через `asyncpg`.
- **REST API** — POST-запрос на `POST /api/v1/news` с токеном администратора.
- **Очередь задач** — через Celery/Redis для асинхронной обработки.

## Зависимости (рекомендуемые)

```
httpx          # HTTP клиент
beautifulsoup4 # парсинг HTML
lxml           # быстрый парсер
playwright     # для JS-рендеримых страниц
apscheduler    # планировщик задач
fake-useragent # ротация User-Agent
```

## Важно

- Соблюдайте `robots.txt` и `Terms of Service` каждого сайта.
- Используйте разумные задержки между запросами (`asyncio.sleep`).
- Не перегружайте серверы источников.
